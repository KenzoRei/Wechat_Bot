# Post-Project Learning List

Topics to study after the project is complete.
Grouped by category. Add new items as they come up during development.

---

## Python & OOP Fundamentals

- [ ] Abstraction and Inheritance in depth — understand ABC, @abstractmethod, method overriding, super()
- [ ] Core OOP principles (the full set): Encapsulation, Abstraction, Inheritance, Polymorphism
- [ ] When to use inheritance vs composition — a common design decision in real projects
- [ ] @property decorator — turns a method into a readable attribute; why it's paired with @abstractmethod in ABCs to enforce subclass contracts

## Web Frameworks

- [ ] How Flask works — minimal framework, routing, request/response cycle
- [ ] How Django works — full-stack framework, ORM, admin panel, when it makes sense

## Backend Fundamentals

- [ ] How database.py works in depth — SQLAlchemy engine, session lifecycle, dependency injection with Depends(get_db), connection pooling

## Database Tools

- [ ] dbdiagram.io — learn the DBML syntax to draw ER diagrams from scratch, not just import

---

## Dev Tools

- [ ] ngrok — what it does (exposes local server to internet via tunnel), how authtoken works, free vs paid limits, why the URL changes every session on free plan

---

## How to use this list

After the project ships, pick one topic at a time.
Build a small example project for each — reading alone is not enough.
